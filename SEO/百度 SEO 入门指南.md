## SEO 定义与分类

> 来源：《百度搜索引擎优化指南2.0》（2014-12-12）

**搜索引擎优化**（Search Engine Optimization，简称 SEO）指为了提升网页在搜索引擎自然搜索结果中（非商业性推广结果）的收录数量以及排序位置而做的优化行为，目的是从搜索引擎中获得更多免费流量以及更好的展现形象。

**SEM**（Search Engine Marketing，搜索引擎营销）既包括 SEO，也包括付费的商业推广优化。

SEO 行为分为两类：

- **白帽 SEO**：改良和规范网站设计，使之对搜索引擎和用户更加友好，获取更多合理流量。搜索引擎鼓励和支持此类 SEO
- **黑帽 SEO**：利用和放大搜索引擎策略缺陷获取更多用户访问量，以伤害用户体验为代价。搜索引擎会通过策略遏制此类行为

搜索引擎与 SEO 是良性的共生关系。了解 SEO 基本原理后，通过合理优化使优质资源发挥更好检索效果，改善用户搜索体验。

## 百度搜索资源平台工具

> 来源：百度搜索资源平台最新文档（2024-04-26 更新）

百度搜索资源平台（ziyuan.baidu.com）是百度官方提供的站长服务平台，核心功能包括：

### 快速抓取

> 2024年4月26日上线，替代原有的"快速收录"工具

**功能说明**：
- 站长可通过快速抓取工具向百度 spider 主动推送资源
- 缩短爬虫发现网站链接的时间
- 提交的资源将**优先抓取**
- 对是否收录及线上展现效果**不保证**

**提交方式**：
- 手动提交：在平台直接输入 URL
- API 提交：使用平台提供的接口推送

**注意事项**：
- 提交资源即占用配额，请谨慎选择提交方式
- 建议先推送少量资源，关注推送反馈状态码，成功后可批量提交
- 如果网站已通过 HTTPS 认证工具验证为 HTTPS 网站，请确保所提交链接的协议头为 HTTPS
- 平台会定期对站点权益进行复审，不符合百度搜索质量&体验标准，权益将被收回

**权益获取方式**：
- 站点成为搜索资源平台 VIP 合作伙伴，自动赋予权益
- 站点非 VIP，关注百度搜索资源平台活动，通过活动获得权益

### 普通收录

**功能说明**：
- 网站主动向百度搜索推送数据的工具
- 可缩短爬虫发现网站链接时间
- 网站时效性内容建议使用此工具实时推送
- 可加快爬虫抓取速度，**无法解决网站内容是否收录问题**

**注意**：百度搜索资源平台为站长提供链接提交通道，可提交想被百度收录的链接，百度搜索引擎会按照标准处理，**但不保证一定能够收录**提交的链接。

### 移动适配

**功能说明**：
- 为提升搜索用户在百度移动搜索的检索体验，会给对应 PC 页面的手机页面在搜索结果处有更多的展现机会
- 需要站点向百度提交主体内容相同的 PC 页面与移动页面的对应关系

**注意事项**：
- **自适应站点不需要使用移动适配工具**
- 移动适配工具**不能解决移动端排序问题**，搜索排序异常请在反馈中心提交
- 移动适配工具校验时间为 **14 天**，如数据提交14天后仍未校验完成，请在反馈中心提交问题

### 死链提交

用于提交网站中已失效的链接，帮助百度及时清理搜索结果中的死链。

### HTTPS 认证

> 百度已支持 HTTPS 网站收录

百度搜索资源平台提供 HTTPS 认证工具，已验证为 HTTPS 网站的站点可正常收录。使用快速抓取或普通收录提交链接时，需确保链接协议头为 HTTPS。

**注意**：2014年指南中关于"百度只能收录少部分 https 网页"的内容已过时，当前百度已正常支持 HTTPS 网站收录。

### Robots 工具

> 2018年9月升级

Robots 是站点与 spider 沟通的重要渠道，站点通过 robots 文件声明本网站中不想被搜索引擎收录的部分或者指定搜索引擎只收录特定的部分。

**robots.txt 文件格式**：

- `User-agent:` 描述搜索引擎 robot 的名字。设为 `*` 则对任何 robot 均有效
- `Disallow:` 描述不希望被访问的 URL
- `Allow:` 描述希望被访问的 URL
- 支持使用通配符 `*`（匹配0或多个任意字符）和 `$`（匹配行结束符）

**百度会严格遵守 robots 相关协议**，请注意区分不想被抓取或收录的目录的大小写，百度会对 robots 中所写的文件和不想被抓取和收录的目录做精确匹配，否则 robots 协议无法生效。

**视频资源 robots 说明**（2018年9月升级）：
- 如网站未设置 robots 协议，百度搜索对网站视频 URL 的收录将包含视频播放页 URL 及页面中的视频文件、视频周边文本等信息
- 已收录的短视频资源将对用户呈现为**视频极速体验页**
- 综艺影视类长视频，搜索引擎仅收录页面 URL

## 前期准备

> 来源：《百度搜索引擎优化指南2.0》（2014-12-12）

### 域名注册

域名是用户对网站的第一印象，能否让用户迅速记住域名对网站发展非常重要。

建议：

- 域名尽量简短，降低用户记忆成本
- 域名可以和网站主题或名称相呼应，使用公司名称、商标、网站名称或产品等信息选择域名
- 域名后缀形式对百度搜索没有影响，但 `.com`、`.cn`、`.net` 等常见后缀用户更容易记忆

### 服务器、空间租用

服务器、空间的速度和稳定性直接影响用户体验，也会影响搜索引擎抓取。

建议：

- 选择有实力的正规空间商。无实力的空间商可能无法提供稳定服务，甚至可能故意屏蔽 Baiduspider 抓取
- 保证用户访问速度和稳定性
- 根据网站规模和服务需求选择合适空间（服务器）
- 根据用户群分布选择接入商

## 面向搜索引擎的网站建设

> 来源：《百度搜索引擎优化指南2.0》（2014-12-12），基础原则仍然适用

搜索引擎只是网站的普通访客，对网站的抓取方式和价值判断都从用户角度出发。任何用户体验改进都是对搜索引擎的改进。

主要分为三部分：**收录、排序、展现**。

### 良好收录

#### 机器可读

百度通过 **Baiduspider** 程序抓取互联网网页，处理后建入索引。Baiduspider 目前只能读懂文本内容，flash、图片等非文本内容暂时不能处理。

建议：

- 使用文字而不是 flash、图片、Javascript 显示重要内容或链接
- Ajax 等搜索引擎不能识别的技术仅用于需要用户交互的地方，不把希望搜索引擎"看"到的导航及正文内容放到 Ajax 中
- 不使用 frame 和 iframe 框架结构，通过 iframe 显示的内容可能会被百度丢弃

#### 网站结构

网站应有清晰结构和明晰导航。

**树型结构**分为三个层次：首页——频道——文章页。理想结构应更扁平，从首页到内容页层次尽量少。

同时网站应是**网状结构**，每个网页都应有指向上、下级网页及相关内容的链接。

合理的网站结构应为**扁平的树型网状结构**。建议：

- 确保每个页面都可以通过至少一个文本链接到达
- 重要内容应能从首页或较浅层次访问到
- 合理分类网站内容，不要过度细分

**导航建议**：

- 每个页面都加上导航栏，方便用户返回频道和首页
- 内容较多的网站建议使用面包屑导航
- 导航中使用文字链接，不使用复杂的 JS 或 Flash

#### 规范、简单的 URL

建议：

- **同一网页只对应一个 URL**。多种 URL 形式会导致：搜索引擎选一种为标准；用户为同一网页不同 URL 做推荐，分散网页权重
- 让用户能从 URL 判断网页内容及网站结构信息
- URL 尽量短
- **正常动态 URL 对搜索引擎没有影响**。建议减少动态 URL 中变量参数

#### 改版/换域名

网站改版、换域名处理不当会造成重大流量损失。

建议：

- 改版或换域名时，将旧网页 **301 永久重定向**到内容对应的新网页
- 如非必要，不做整站内容完全更换
- 网站更换域名，旧域名所有页面 301 到新域名对应页面
- 改版/换域名后，通过 sitemap 提交新 URL 给百度

#### 合理的返回码

百度爬虫根据 HTTP 协议规范设置逻辑。常用返回码处理逻辑：

| 返回码 | 含义 | 百度处理 |
|--------|------|----------|
| 404 | NOT FOUND | 认为网页已失效，从搜索结果删除，短期内再次发现也不抓取 |
| 503 | Service Unavailable | 认为网页临时不可访问，不直接删除，短期内再访问 |
| 403 | Forbidden | 认为网页当前禁止访问，新发现 URL 暂不抓取 |
| 301 | Moved Permanently | 认为网页跳转至新 URL，站点迁移、域名更换、改版时推荐使用 |

建议：

- 站点临时关闭，网页不能打开时不要立即返回 404，建议使用 **503**
- 百度抓取压力过大时建议返回 503
- 站点迁移或域名更换时使用 **301**

### 良好排序

#### 涵盖网页主要内容的 title

网页 title 用于告诉用户和搜索引擎网页主要内容，是判断网页内容权重的主要参考信息之一。

网页标题描述建议：

- 首页：网站名称 或 网站名称_提供服务介绍或产品介绍
- 频道页：频道名称_网站名称
- 文章页：文章title_频道名称_网站名称

推荐做法：

- 每个网页应有独一无二的标题
- 标题主题明确，包含网页最重要内容
- 简明精练，不罗列与网页内容不相关信息
- 重要内容放到 title 靠前位置
- 使用用户熟知语言描述

#### 良好的内容建设

- 网站内容建设以服务网站核心价值为主，内容应符合网站主题
- 内容应面向用户，提供符合用户需求的原创内容至关重要
- **写好锚文本**：锚文本是做链接时的描述文字
- **为图片加 alt 说明**
- 管理 Web2.0 等用户产生内容的产品，被作弊者利用可能影响整个站点权重

#### 赢得用户对网站的推荐

互联网上提供相同内容、服务的网站很多，内容相同时，哪个网站排在前面取决于**推荐**。

超链是推荐的一种。用户推荐形式多样：即时通讯工具发给朋友、论坛转帖推荐、友情链接等。

### 良好展现

#### 吸引眼球的 Title

用户搜索到网页时，title 作为最重要内容显示在摘要中。

推荐做法：

- 标题主题明确，包含网页最重要内容
- 使用用户熟知语言描述
- 网站用户比较熟时，建议将网站名称列到 title 合适位置
- 标题要对用户有吸引力，让用户产生信任感

#### 善用 Meta description

Meta description 是对网页内容的精练概括。

**注意**：Meta description **不是权值计算参考因素**，只做搜索结果摘要选择目标。

推荐做法：

- 网站首页、频道页、产品参数页等最适合使用 description
- 准确描述网页，不要堆砌关键词
- 为每个网页创建不同的 description

## 网站运营

> 来源：《百度搜索引擎优化指南2.0》（2014-12-12）

### 目标=转化率

从搜索引擎获取流量的最终目的是不断提高网站核心价值。从搜索引擎获得的流量有多少能转化为网站核心价值，这就是**转化率**。

建议将转化率列为最重要的衡量效果指标。

### 搜索引擎流量分析

分析指标：

- **跳出率**：只浏览一页便离开的用户比例
- **退出率**：用户从某个页面离开次数占总浏览量比例
- **用户停留时间**：反映网站粘性

### 网站信任度

建议：

- 页面美观、整洁，有自己的风格
- 详细网站介绍、联系方式
- 用户评论、顾客反馈等信息

## 作弊与惩罚

> 来源：《百度搜索引擎优化指南2.0》（2014-12-12）及最新算法违规解读（2021-01-15）

### 百度如何定义作弊

任何利用和放大搜索引擎策略缺陷，利用恶意手段获取与网页质量不符的排名，引起搜索结果质量和用户搜索体验下降的行为都会被搜索引擎当做作弊行为。

基本界定法则：这个行为的泛滥是否会影响搜索系统，最终伤害用户搜索体验。

### 作弊会受到怎样的惩罚

处理手段始终维持"轻者轻罚，重者重罚"原则：

- 对用户体验及搜索结果质量影响不大的：去除作弊部分获得的权值
- 对用户体验及搜索结果质量影响严重的：去除作弊部分获得的权值并降低网站权重，直至从搜索结果彻底清理

### 改正后能否解除惩罚

惩罚不是目的，让互联网洁净才是目的。有完善流程，定期自动对作弊网站检测，大部分修正了作弊行为的网站会在一定观察期后自动解除惩罚。

## 常见问题（Q&A）

> 来源：《百度搜索引擎优化指南2.0》（2014-12-12），部分已标注更新

### 百度商业推广和自然搜索结果的关系

百度的商业推广和自然搜索由完全独立的两个部门分别运营两套独立系统。参加商业推广的网站在自然结果中一视同仁，没有任何特殊处理。

商业推广原理不是"给了钱，自然结果某些结果就可以排得更靠前"，而是"用户关键词被分发到两个独立系统，分别产生商业结果和自然结果，商业结果在前，自然结果在后"。

### 百度是否支持 nofollow

百度支持 `<a rel="nofollow" href="url">` 和 `<meta name="robots" content="nofollow">` 两种写法的 nofollow。带有 nofollow 属性的 URL 不会传递权值。

### 百度是否支持 HTTPS 协议

> **已更新**：百度已正常支持 HTTPS 网站收录

百度搜索资源平台提供 HTTPS 认证工具，已验证的 HTTPS 网站可正常收录。使用资源提交工具时需确保链接协议头为 HTTPS。

**2014年指南原文（已过时）**："百度目前只能收录少部分 https 网页，大部分 https 网页无法收录。"——此内容已不再适用。

### Site 语法查到的结果数是否为百度收录网页数量

site 语法得到的搜索结果数只是一个估算数值，仅供参考。

### 修改网站标题是否会对排名带来消极影响

title 是极重要的内容。大幅修改可能会带来大幅波动。建议按推荐写法实事求是将页面主旨反映在标题中。

### 百度是否建议 URL 静态化

URL 是动态还是静态对百度没有影响，没有任何歧视动态 URL 的策略。

建议尽量减少动态 URL 中包含的变量参数。

### 百度支持哪些 Robots Meta 标签

百度支持 **nofollow**、**noarchive** 两种 meta 标签。

- 防止所有搜索引擎显示网站快照：`<meta name="robots" content="noarchive">`
- 不想搜索引擎追踪此网页链接且不传递链接权重：`<meta name="robots" content="nofollow">`

### 站点启用 CDN、反向代理、gzip 等服务是否影响搜索引擎收录

Baiduspider 对站点的抓取方式和普通用户访问一样，只要普通用户能访问到的内容就能抓取到。

建议尽量选择有实力的服务商和成熟的技术。

### 百度对使用海外空间的中文网站有歧视吗

百度对海外站点没有歧视。但 Baiduspider 服务器在国内，国内普通用户访问受限的内容，Baiduspider 一样会搞不定。只要国内能正常访问，就会一视同仁。
