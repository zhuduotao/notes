## 概述

JavaScript 的数据类型分为两大类：**基本类型（Primitive Types）** 和 **引用类型（Reference Types）**。理解它们的存储方式、行为特征和底层机制，是掌握 JavaScript 的核心基础。

---

## 一、七种基本类型

### 1. `undefined`

- **含义**：变量已声明但未赋值时的默认值
- **类型检测**：`typeof undefined === 'undefined'`
- **常见场景**：
  - 变量声明未初始化
  - 函数无返回值时默认返回 `undefined`
  - 对象属性不存在时访问返回 `undefined`
  - 函数参数未传递时为 `undefined`

```typescript
let a;
console.log(a); // undefined

function test() {}
console.log(test()); // undefined

const obj = {};
console.log(obj.nonExist); // undefined
```

### 2. `null`

- **含义**：表示"空值"或"不存在的对象"
- **类型检测**：`typeof null === 'object'`（这是 JavaScript 的历史 bug）
- **与 undefined 的区别**：
  - `undefined` 表示"未定义"
  - `null` 表示"有意设置为空"
- **最佳实践**：手动清空对象引用时使用 `null`

```typescript
let element = document.getElementById('myDiv');
// 使用完毕后
element = null; // 释放引用，帮助 GC 回收
```

### 3. `boolean`

- **取值**：`true` 或 `false`
- **类型检测**：`typeof true === 'boolean'`
- **隐式转换**：在条件判断中会自动转换为布尔值

#### 假值（Falsy Values）

以下值在布尔上下文中会被转换为 `false`：

- `false`
- `0` / `-0` / `0n`
- `""` / `''` / `` `` ``（空字符串）
- `null`
- `undefined`
- `NaN`

```typescript
if (0) console.log('不会执行');
if ('') console.log('不会执行');
if ([]) console.log('会执行，空数组是真值');
if ({}) console.log('会执行，空对象是真值');
```

### 4. `number`

- **含义**：表示整数和浮点数（IEEE 754 双精度 64 位格式）
- **类型检测**：`typeof 42 === 'number'`
- **特殊值**：
  - `NaN`（Not-a-Number）
  - `Infinity` / `-Infinity`
  - `Number.MAX_SAFE_INTEGER`（2^53 - 1）
  - `Number.MIN_SAFE_INTEGER`（-(2^53 - 1)）

```typescript
console.log(typeof NaN); // 'number'
console.log(NaN === NaN); // false，必须用 Number.isNaN() 判断
console.log(0.1 + 0.2 === 0.3); // false，浮点数精度问题
console.log(Number.EPSILON); // 2.220446049250313e-16，用于精度比较
```

#### 精度问题解决方案

```typescript
// 方案 1：使用 toFixed
(0.1 + 0.2).toFixed(1) === '0.3';

// 方案 2：乘以倍数转整数
Math.round((0.1 + 0.2) * 10) / 10 === 0.3;

// 方案 3：使用 Number.EPSILON
Math.abs(0.1 + 0.2 - 0.3) < Number.EPSILON;
```

### 5. `bigint`

- **含义**：表示任意精度的整数（ES2020 引入）
- **类型检测**：`typeof 1n === 'bigint'`
- **创建方式**：
  - 在整数后加 `n`：`123n`
  - 使用 `BigInt()` 函数：`BigInt(123)` 或 `BigInt("123")`
- **用途**：解决 `number` 类型超出安全整数范围时的精度丢失问题

```typescript
const big = 9007199254740991n;
console.log(big + 1n); // 9007199254740992n

// number 超出安全范围会丢失精度
console.log(9007199254740991 + 1); // 9007199254740992
console.log(9007199254740991 + 2); // 9007199254740992（精度丢失）
```

### 6. `string`

- **含义**：表示文本数据（UTF-16 编码的不可变序列）
- **类型检测**：`typeof 'hello' === 'string'`
- **不可变性**：字符串一旦创建就不能修改，所有字符串方法都返回新字符串

```typescript
let str = 'hello';
str[0] = 'H'; // 无效，字符串不可变
console.log(str); // 'hello'

str = 'Hello'; // 重新赋值，创建新字符串
```

#### 模板字符串（ES6）

```typescript
const name = 'World';
const greeting = `Hello, ${name}!`;
const multiline = `
  第一行
  第二行
`;
```

### 7. `symbol`

- **含义**：表示独一无二的值（ES6 引入）
- **类型检测**：`typeof Symbol() === 'symbol'`
- **特性**：
  - 每次调用 `Symbol()` 都返回唯一值
  - 可作为对象属性名，避免属性名冲突
  - 不会被 `for...in`、`Object.keys()` 遍历

```typescript
const s1 = Symbol('desc');
const s2 = Symbol('desc');
console.log(s1 === s2); // false

const obj = {
  [s1]: 'value1',
  [s2]: 'value2',
};

console.log(Object.keys(obj)); // []，Symbol 属性不可枚举
console.log(Object.getOwnPropertySymbols(obj)); // [Symbol(desc), Symbol(desc)]
```

#### Symbol.for() 与 Symbol.keyFor()

```typescript
const s1 = Symbol.for('shared');
const s2 = Symbol.for('shared');
console.log(s1 === s2); // true，全局符号注册表复用

console.log(Symbol.keyFor(s1)); // 'shared'
```

## 二、基本类型的存储机制

### 栈内存（Stack）

基本类型的值直接存储在**栈内存**中：

```typescript
let a = 10;
let b = a; // 拷贝值
b = 20;

console.log(a); // 10，互不影响
console.log(b); // 20
```

**特点**：
- 占用空间小，大小固定
- 按值访问，访问速度快
- 变量赋值时创建副本，互不影响

## 三、类型检测

### typeof 操作符

```typescript
typeof undefined;    // 'undefined'
typeof null;         // 'object'（历史 bug）
typeof true;         // 'boolean'
typeof 42;           // 'number'
typeof 42n;          // 'bigint'
typeof 'hello';      // 'string'
typeof Symbol();     // 'symbol'
typeof {};           // 'object'
typeof [];           // 'object'
typeof function(){}; // 'function'
```

### Object.prototype.toString.call()

最准确的类型检测方法：

```typescript
const typeCheck = (value: unknown): string => {
  return Object.prototype.toString.call(value).slice(8, -1);
};

typeCheck(undefined); // 'Undefined'
typeCheck(null);      // 'Null'
typeCheck(true);      // 'Boolean'
typeCheck(42);        // 'Number'
typeCheck(42n);       // 'BigInt'
typeCheck('hello');   // 'String'
typeCheck(Symbol());  // 'Symbol'
typeCheck({});        // 'Object'
typeCheck([]);        // 'Array'
typeCheck(/regex/);   // 'RegExp'
typeCheck(new Date());// 'Date'
typeCheck(new Map()); // 'Map'
typeCheck(new Set()); // 'Set'
```

## 四、类型转换

### 隐式转换规则

JavaScript 在运算时会进行隐式类型转换，理解转换规则是避免 bug 的关键。

#### 转字符串（ToString）

```typescript
String(123);        // '123'
String(null);       // 'null'
String(undefined);  // 'undefined'
String(true);       // 'true'
String([]);         // ''
String({});         // '[object Object]'
```

#### 转数字（ToNumber）

```typescript
Number('123');      // 123
Number('');         // 0
Number('  ');       // 0
Number(null);       // 0
Number(undefined);  // NaN
Number(true);       // 1
Number(false);      // 0
Number([]);         // 0
Number({});         // NaN
```

#### 转布尔值（ToBoolean）

规则：只有假值转为 `false`，其余全部转为 `true`。

### 常见陷阱

```typescript
// 字符串拼接 vs 数字相加
console.log(1 + '2');      // '12'，字符串拼接
console.log(1 + 2);        // 3，数字相加
console.log('1' - '2');    // -1，减法会转数字

// 相等比较
console.log(0 == false);   // true
console.log(0 === false);  // false
console.log('' == 0);      // true
console.log('' === 0);     // false
console.log(null == undefined); // true
console.log(null === undefined); // false

// 推荐使用严格相等
console.log(Object.is(NaN, NaN)); // true
console.log(Object.is(0, -0));    // false
```

## 五、包装对象（Wrapper Objects）

基本类型在访问属性或方法时，JavaScript 会临时创建对应的包装对象：

```typescript
const str = 'hello';
console.log(str.toUpperCase()); // 'HELLO'

// 等价于：
const temp = new String(str);
const result = temp.toUpperCase();
// temp 立即被销毁
```

三种包装对象：
- `String`
- `Number`
- `Boolean`

```typescript
const s = new String('hello');
const n = new Number(42);
const b = new Boolean(true);

typeof s; // 'object'
typeof n; // 'object'
typeof b; // 'object'

// 不推荐显式创建包装对象
const primStr = 'hello';
const objStr = new String('hello');

console.log(typeof primStr); // 'string'
console.log(typeof objStr);  // 'object'
console.log(primStr == objStr);  // true
console.log(primStr === objStr); // false
```

## 六、最佳实践

### 1. 类型检测优先使用严格相等

```typescript
// ❌ 不推荐
if (value == null) { }

// ✅ 推荐
if (value === null || value === undefined) { }
// 或
if (value == null) { } // 仅在明确知道要同时判断 null 和 undefined 时
```

### 2. 使用 Number.isNaN 判断 NaN

```typescript
// ❌ 不推荐
isNaN(value); // 会先转数字，可能误判

// ✅ 推荐
Number.isNaN(value); // 只对 NaN 返回 true
```

### 3. 使用 Number.isFinite 判断有效数字

```typescript
// ❌ 不推荐
isFinite(value); // 会先转数字

// ✅ 推荐
Number.isFinite(value); // 不会隐式转换
```

### 4. 大数使用 BigInt

```typescript
// 超出安全整数范围时使用 BigInt
const bigId = 9007199254740993n;
```

### 5. 避免隐式类型转换陷阱

```typescript
// 明确转换类型
const num = Number(str);
const str = String(num);
const bool = Boolean(value);

// 或使用一元运算符
const num = +str;
```
