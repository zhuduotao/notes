## 前置要求

### 系统依赖

```bash
# macOS
brew install git python3

# Ubuntu/Debian
sudo apt-get install git python3 build-essential
```

### depot_tools

```bash
# 克隆 depot_tools
git clone https://chromium.googlesource.com/chromium/tools/depot_tools.git
cd depot_tools
export PATH="$PWD:$PATH"

# 添加到 ~/.zshrc 或 ~/.bashrc
echo 'export PATH="/path/to/depot_tools:$PATH"' >> ~/.zshrc
```

## 获取源码

```bash
# 创建 V8 目录
mkdir v8 && cd v8

# 获取 V8 源码
fetch v8

# 或只获取特定分支
fetch v8 --branch=12.0
```

## 编译配置

### 基础编译参数

```bash
# 进入 v8 目录
cd v8

# 生成构建配置（Release）
tools/dev/v8gen.py x64.release

# 生成构建配置（Debug，包含调试符号）
tools/dev/v8gen.py x64.debug

# 查看可用配置
tools/dev/v8gen.py list
```

### 自定义 GN 参数

```bash
# 编辑构建参数
gn args out.gn/x64.debug
```

常用参数：

```gn
# 调试模式
is_debug = true

# 启用调试符号
symbol_level = 2

# 禁用优化（便于调试）
v8_enable_backtrace = true
v8_enable_slow_dchecks = true
v8_optimized_debug = false

# 启用 V8 内部调试工具
v8_enable_object_print = true
v8_enable_verify_heap = true

# 编译静态库（便于链接）
is_component_build = false
v8_monolithic = true
v8_use_external_startup_data = false
```

## 编译

```bash
# 使用 ninja 编译
ninja -C out.gn/x64.debug

# 或使用自动脚本
tools/dev/v8gen.py x64.debug
ninja -C out.gn/x64.debug
```

## 调试配置

### LLDB 调试

```bash
# 启动 LLDB
lldb -- ./out.gn/x64.debug/d8 script.js

# 或使用 alias
alias v8dbg='lldb -- ./out.gn/x64.debug/d8'
```

常用 LLDB 命令：

```lldb
# 设置断点
breakpoint set --name FunctionName
breakpoint set --file source.cc --line 123

# V8 对象打印
command script import tools/lldb_commands.py
v8 findjsobjects
v8 findjsinstances Array
v8 inspect 0x123456789

# 运行
run script.js
continue
next
step
```

### GDB 调试

```bash
# 启动 GDB
gdb --args ./out.gn/x64.debug/d8 script.js

# V8 GDB 脚本
source tools/gdbinit
```

### VSCode 调试配置

`.vscode/launch.json`:

```json
{
  "version": "0.2.0",
  "configurations": [
    {
      "name": "Debug V8 d8",
      "type": "lldb",
      "request": "launch",
      "program": "${workspaceFolder}/out.gn/x64.debug/d8",
      "args": ["${file}"],
      "cwd": "${workspaceFolder}",
      "preLaunchTask": "build debug",
      "initCommands": [
        "command script import ${workspaceFolder}/tools/lldb_commands.py"
      ]
    }
  ]
}
```

## 常见编译场景

### 编译独立可执行文件

```gn
# 在 gn args 中添加
is_component_build = false
v8_monolithic = true
```

```bash
ninja -C out.gn/x64.debug d8
```

### 编译静态库

```bash
# 编译 libv8_monolith.a
ninja -C out.gn/x64.debug v8_monolith
```

### 编译特定组件

```bash
# 只编译 d8
ninja -C out.gn/x64.debug d8

# 编译所有测试
ninja -C out.gn/x64.debug cctest

# 编译 mksnapshot
ninja -C out.gn/x64.debug mksnapshot
```

## 调试技巧

### V8 Flags

```bash
# 启用详细日志
./out.gn/x64.debug/d8 --trace-opt --trace-deopt script.js

# 打印生成的代码
./out.gn/x64.debug/d8 --print-opt-code script.js

# TurboFan 追踪
./out.gn/x64.debug/d8 --trace-turbo script.js

# 查看所有可用 flags
./out.gn/x64.debug/d8 --help
```

### 断点调试

```cpp
// 在源码中添加断点
#include "src/base/logging.h"
DCHECK(false);  // 触发断点
```

### 内存调试

```bash
# 启用内存检查
./out.gn/x64.debug/d8 --verify-heap script.js

# 垃圾回收日志
./out.gn/x64.debug/d8 --trace-gc script.js
```

## 常见问题

### 1. 编译失败：缺少依赖

```bash
# 重新获取依赖
gclient sync
```

### 2. 调试符号缺失

确保 `symbol_level = 2` 且 `is_debug = true`

### 3. 编译太慢

```gn
# 使用并行编译
ninja -C out.gn/x64.debug -j$(nproc)

# 或使用 ccache
export CC_WRAPPER=ccache
export CXX_WRAPPER=ccache
```

### 4. LLDB 命令不工作

```bash
# 确保使用 debug 版本
# 确保导入了 lldb_commands.py
command script import /path/to/v8/tools/lldb_commands.py
```
