## Chrome DevTools 调试

### 基础调试命令

```javascript
// 设置断点
debugger;

// 控制台 API
console.log()      // 普通日志
console.debug()    // 调试日志
console.info()     // 信息日志
console.warn()     // 警告日志
console.error()    // 错误日志
console.table()    // 表格形式输出对象数组
console.trace()    // 输出调用栈
console.time()     // 计时开始
console.timeEnd()  // 计时结束
console.count()    // 计数
console.group()    // 分组开始
console.groupEnd() // 分组结束
```

### Performance 面板

1. **Performance 录制**
   - 点击录制按钮开始
   - 执行需要分析的操作
   - 停止录制查看结果

2. **关键指标**
   - FP (First Paint): 首次绘制
   - FCP (First Contentful Paint): 首次内容绘制
   - LCP (Largest Contentful Paint): 最大内容绘制
   - TTI (Time to Interactive): 可交互时间
   - CLS (Cumulative Layout Shift): 累积布局偏移

3. **火焰图分析**
   - 黄色：JavaScript 执行
   - 紫色：样式计算与布局
   - 绿色：绘制与栅格化
   - 蓝色：解析 HTML
   - 浅蓝：脚本解析与编译

### Memory 面板

1. **堆快照 (Heap Snapshot)**
   - 拍摄当前内存状态
   - 对比多个快照查找内存泄漏
   - 查看对象引用链

2. **分配时间线 (Allocation Timeline)**
   - 实时记录内存分配
   - 识别内存增长模式

3. **分配采样 (Allocation Sampling)**
   - 低开销的内存分析
   - 适合生产环境

### V8 内部调试

#### 1. V8 标志 (Flags)

```bash
# 启动 Node.js 或 Chrome 时添加标志
--js-flags="--allow-natives-syntax"  # 允许 %FunctionName 语法
--js-flags="--trace-opt"             # 追踪优化
--js-flags="--trace-deopt"           # 追踪去优化
--js-flags="--print-opt-code"        # 打印优化后的代码
--js-flags="--trace-turbo"           # 追踪 TurboFan 编译器
```

#### 2. V8 内置函数 (Natives Syntax)

```javascript
// 需要 --allow-natives-syntax 标志

// 函数优化状态
%GetOptimizationStatus(fn)           // 获取优化状态
%OptimizeFunctionOnNextCall(fn)      // 下次调用时优化
%DeoptimizeFunction(fn)              // 去优化函数
%NeverOptimizeFunction(fn)           // 永不优化

// 类型信息
%MapOf(obj)                          // 获取对象 Map
%HasFastProperties(obj)              // 是否有快速属性
%HasDictionaryMode(obj)              // 是否字典模式

// 内联缓存
%ICStatus(fn, slot)                  // 获取 IC 状态

// 其他
%FunctionName(fn)                    // 函数名
%FunctionCloneCount(fn)              // 克隆次数
%SystemBreak()                       // 系统断点
```

#### 3. 优化状态码

```javascript
// %GetOptimizationStatus 返回值
1     // 从未优化
2     // 已优化
4     // 始终优化
8     // 执行优化中
16    // 标记为同时优化
32    // 标记为再次优化
64    // 标记为优化后
128   // 优化失败
256   // 标记为内联
512   // 标记为内联到调用者
```

### Node.js 调试

#### 1. Inspector API

```javascript
const inspector = require('inspector');

// 打开 inspector
inspector.open(9229, 'localhost', true);

// 等待调试器连接
inspector.waitForDebugger();

// 关闭
inspector.close();
```

#### 2. 命令行调试

```bash
# 启动调试模式
node --inspect app.js
node --inspect=9229 app.js
node --inspect-brk app.js  # 第一行断点

# 使用调试器
node inspect app.js
```

#### 3. 性能分析

```bash
# CPU 分析
node --prof app.js
node --prof-process isolate-*.log

# 堆分析
node --trace-gc app.js
node --heapsnapshot-signal=SIGUSR2 app.js
```

### 常见优化问题

#### 1. 隐藏类 (Hidden Classes)

```javascript
// ❌ 导致不同隐藏类
function Point(x, y) {
  this.x = x;
  this.y = y;
}
const p1 = new Point(1, 2);
const p2 = new Point(1, 2);
p2.z = 3;  // 创建新隐藏类

// ✅ 保持相同隐藏类
function Point(x, y) {
  this.x = x;
  this.y = y;
  this.z = undefined;  // 预定义属性
}
```

#### 2. 内联缓存 (Inline Caches)

```javascript
// ❌ 多态调用，IC 失效
function add(a, b) {
  return a + b;
}
add(1, 2);        // 数字
add('a', 'b');    // 字符串
add({}, {});      // 对象

// ✅ 单态调用，IC 命中
function add(a, b) {
  return a + b;
}
add(1, 2);
add(3, 4);
add(5, 6);
```

#### 3. 去优化 (Deoptimization)

```javascript
// ❌ 类型变化导致去优化
function sum(arr) {
  let total = 0;
  for (let i = 0; i < arr.length; i++) {
    total += arr[i];  // 假设都是数字
  }
  return total;
}
sum([1, 2, 3]);
sum([1, '2', 3]);  // 去优化

// ✅ 类型守卫
function sum(arr) {
  let total = 0;
  for (let i = 0; i < arr.length; i++) {
    if (typeof arr[i] !== 'number') {
      throw new Error('Expected number');
    }
    total += arr[i];
  }
  return total;
}
```

### 调试技巧

1. **使用 Performance.now()** 替代 Date.now() 进行精确计时
2. **使用 console.time/timeEnd** 进行代码块性能测试
3. **使用 Performance 面板的火焰图** 识别性能瓶颈
4. **使用 Memory 面板对比快照** 查找内存泄漏
5. **使用 --trace-opt 和 --trace-deopt** 分析优化行为
6. **避免 eval() 和 with** 阻止优化
7. **保持函数单态** 提高 IC 命中率
8. **预定义对象属性** 避免隐藏类变化
