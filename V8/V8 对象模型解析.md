## 概述

V8 使用一套复杂的对象模型来高效存储和访问 JavaScript 对象，核心概念包括隐藏类 (Maps)、属性存储和元素存储。

## JSObject 内存布局

```
┌─────────────────────────────────┐
│         JSObject Header         │
├─────────────────────────────────┤
│  Map Pointer (隐藏类指针)        │
│  Properties Pointer (属性)       │
│  Elements Pointer (元素)         │
│  In-object Properties (内联属性)│
└─────────────────────────────────┘
```

## 隐藏类 (Maps / Hidden Classes)

### 什么是隐藏类

隐藏类是 V8 用来跟踪对象形状的元数据结构：

- 属性名称
- 属性顺序
- 属性偏移量
- 原型链信息

### 为什么需要隐藏类

```javascript
// JavaScript 是动态语言，对象可以随时改变形状
const obj = {};
obj.x = 1;        // 形状 1
obj.y = 2;        // 形状 2
delete obj.x;     // 形状 3
```

V8 通过隐藏类将动态语言转换为类似静态类型的访问模式。

### 隐藏类转换

```javascript
const obj = {};
// Map A: {}

obj.x = 1;
// Map B: {x} → 从 Map A 转换而来

obj.y = 2;
// Map C: {x, y} → 从 Map B 转换而来
```

转换关系形成树状结构：

```
Map A ({})
   │
   ├─ 添加 x → Map B ({x})
   │              │
   │              └─ 添加 y → Map C ({x, y})
   │
   └─ 添加 y → Map D ({y})
                  │
                  └─ 添加 x → Map E ({y, x})
```

**注意**: Map C 和 Map E 虽然属性相同，但顺序不同，是不同的隐藏类。

## 属性存储 (Properties)

### 存储位置

V8 将属性存储在两个位置：

1. **In-object Properties** - 存储在对象内部
2. **Out-of-object Properties** - 存储在外部属性数组

### In-object Properties

```javascript
const obj = { x: 1, y: 2 };
// x 和 y 直接存储在对象内部，访问最快
```

特点：
- 访问速度最快（无需额外指针跳转）
- 数量有限（受对象头大小限制）
- 对象创建时确定

### Out-of-object Properties

```javascript
const obj = {};
obj.a = 1;
obj.b = 2;
obj.c = 3;
// 超出内联容量的属性存储在外部数组
```

特点：
- 通过 Properties Pointer 访问
- 可以动态扩展
- 访问稍慢（需要额外指针解引用）

### 属性存储转换

```javascript
const obj = { x: 1 };
// In-object: [x]

obj.y = 2;
obj.z = 3;
// In-object: [x, y, z] (如果容量允许)

obj.w = 4;
// In-object: [x, y, z]
// Out-of-object: [w] (超出容量，溢出到外部)
```

## 元素存储 (Elements)

### 数组 vs 对象

```javascript
const arr = [1, 2, 3];    // 使用 Elements 存储
const obj = { 0: 1 };     // 使用 Properties 存储
```

### Elements 类型

V8 根据内容选择最优的 Elements 类型：

| 类型 | 说明 | 示例 |
|------|------|------|
| **PACKED_SMI_ELEMENTS** | 紧凑小整数 | `[1, 2, 3]` |
| **PACKED_DOUBLE_ELEMENTS** | 紧凑浮点数 | `[1.1, 2.2]` |
| **PACKED_ELEMENTS** | 紧凑普通对象 | `[{a:1}, {b:2}]` |
| **HOLEY_SMI_ELEMENTS** | 含空洞小整数 | `[1, , 3]` |
| **HOLEY_DOUBLE_ELEMENTS** | 含空洞浮点数 | `[1.1, , 3.3]` |
| **HOLEY_ELEMENTS** | 含空洞普通对象 | `[1, , "str"]` |

### Elements 转换层级

```
PACKED_SMI_ELEMENTS
        ↓ (添加浮点数)
PACKED_DOUBLE_ELEMENTS
        ↓ (添加对象)
PACKED_ELEMENTS
        ↓ (删除元素/产生空洞)
HOLEY_ELEMENTS
```

**重要**: 转换是单向的，只能向下转换，不能回退。

```javascript
const arr = [1, 2, 3];           // PACKED_SMI
arr.push(4.5);                   // → PACKED_DOUBLE
arr.push({});                    // → PACKED_ELEMENTS
delete arr[1];                   // → HOLEY_ELEMENTS
```

## 原型链处理

### 原型属性查找

```javascript
class Base {
  baseMethod() {}
}

class Derived extends Base {
  derivedMethod() {}
}

const obj = new Derived();
obj.derivedMethod(); // 直接在对象 Map 中找到
obj.baseMethod();    // 通过原型链查找
```

### 原型链优化

V8 使用 **Prototype Validity Cell** 优化原型链查找：

1. 记录原型链是否被修改
2. 如果未修改，缓存查找结果
3. 修改原型时使缓存失效

```javascript
// 缓存有效
obj.method(); // 快速路径

// 修改原型
Base.prototype.newMethod = function() {};
// 缓存失效，需要重新查找

// 之后恢复快速路径
obj.method(); // 新的缓存建立
```

## 字典模式 (Dictionary Mode)

### 何时触发

当对象变得过于动态时，V8 会切换到字典模式：

1. 大量属性删除
2. 使用非标准属性名
3. 属性数量过多

```javascript
const obj = {};
for (let i = 0; i < 1000; i++) {
  obj[`key${i}`] = i;
  if (i % 2 === 0) {
    delete obj[`key${i-1}`]; // 频繁删除触发字典模式
  }
}
```

### 字典模式特点

- 属性存储在哈希表中
- 失去隐藏类优化
- 访问速度显著下降
- 内存占用增加

## 性能优化实践

### 1. 保持一致的初始化顺序

```javascript
// 好 - 相同顺序
function createPoint(x, y) {
  return { x, y };
}

// 差 - 不同顺序
function createPoint(x, y) {
  if (x > 0) return { x, y };
  return { y, x }; // 不同形状！
}
```

### 2. 避免删除属性

```javascript
// 差
delete obj.temp;

// 好
obj.temp = null; // 保持形状不变
```

### 3. 预分配数组

```javascript
// 好 - 预分配
const arr = new Array(1000);
for (let i = 0; i < 1000; i++) {
  arr[i] = i;
}

// 差 - 动态扩展
const arr = [];
for (let i = 0; i < 1000; i++) {
  arr.push(i);
}
```

### 4. 避免混合类型

```javascript
// 好 - 类型一致
const numbers = [1, 2, 3, 4, 5];

// 差 - 混合类型导致 Elements 降级
const mixed = [1, 2, "three", 4, 5];
```

### 5. 避免空洞

```javascript
// 好 - 紧凑数组
const arr = [1, 2, 3, 4, 5];

// 差 - 空洞导致 Holey 类型
const arr = [1, 2, , 4, 5];
```
