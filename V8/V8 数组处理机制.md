## 概述

V8 引擎对 JavaScript 数组进行了高度优化，主要包含以下几种内部表示形式：

## 数组的内部表示

### 1. PACKED_SMI_ELEMENTS
- 仅包含小整数（Small Integers）
- 32 位系统：-2³¹ 到 2³¹-1
- 64 位系统：-2³² 到 2³²-1
- 最高效的存储方式

### 2. PACKED_DOUBLE_ELEMENTS
- 包含浮点数或超出 SMI 范围的整数
- 使用连续的 64 位浮点数数组存储
- 比 SMI 稍慢，但仍然高效

### 3. PACKED_ELEMENTS
- 包含任意类型的元素（对象、字符串、null、undefined 等）
- 使用指针数组存储
- 灵活性最高，性能相对较低

### 4. HOLEY 变体
- HOLEY_SMI_ELEMENTS
- HOLEY_DOUBLE_ELEMENTS
- HOLEY_ELEMENTS
- 数组中存在"空洞"（稀疏数组）时使用
- 性能比 PACKED 版本差

## 快数组与慢数组（Fast vs Slow Elements）

### 快数组（Fast Elements）
- 使用连续内存存储元素
- 通过索引直接访问，时间复杂度 O(1)
- 上述的 PACKED 和 HOLEY 类型都属于快数组
- V8 的默认数组表示方式

### 慢数组（Slow Elements / Dictionary Elements）
- 当数组过于稀疏时，V8 会将快数组转换为慢数组
- 使用哈希表（字典）存储元素
- 触发条件：
  - 数组长度超过 10,000,000
  - 空洞数量超过数组长度的某个比例（约 50%）
  - 大量使用 `delete` 操作
- 访问速度显著下降，失去数组优化优势

```javascript
// 快数组 → 慢数组示例
const arr = new Array(10000000);
arr[0] = 1;
arr[9999999] = 2; // 可能触发转换为慢数组
```

## 数组类型转换（Element Kind Transition）

V8 的数组类型只能单向转换，且只能向更通用的类型转换：

```
PACKED_SMI_ELEMENTS
  → PACKED_DOUBLE_ELEMENTS
    → PACKED_ELEMENTS
      → HOLEY_ELEMENTS
        ↑
      HOLEY_DOUBLE_ELEMENTS
        ↑
      HOLEY_SMI_ELEMENTS
```

**重要规则：**
- 转换是不可逆的
- 向数组添加非整数会将 SMI 数组升级为 DOUBLE 或 ELEMENTS
- 创建空洞会将 PACKED 升级为 HOLEY

## 性能优化建议

### 1. 避免空洞
```javascript
// 慢 - 创建 HOLEY 数组
const arr = [];
arr[0] = 1;
arr[2] = 3; // 索引 1 为空

// 快 - 保持 PACKED
const arr = [1, 2, 3];
```

### 2. 保持类型一致
```javascript
// 慢 - 类型转换
const arr = [1, 2, 3];
arr.push(3.14); // SMI → DOUBLE
arr.push("str"); // DOUBLE → ELEMENTS

// 快 - 类型一致
const arr = [1, 2, 3]; // 纯整数
const arr2 = [1.1, 2.2, 3.3]; // 纯浮点数
```

### 3. 预分配数组
```javascript
// 推荐 - 预分配大小
const arr = new Array(1000);
for (let i = 0; i < 1000; i++) {
  arr[i] = i;
}
```

### 4. 使用类型化数组
```javascript
// 对于数值计算，使用 TypedArray 更快
const arr = new Int32Array(1000);
const arr2 = new Float64Array(1000);
```

## 内联缓存（Inline Caching）

V8 使用内联缓存优化数组访问：
- 记录数组的 Element Kind
- 对同类型数组操作生成优化的机器码
- 类型变化会导致去优化（deoptimization）

## 常见陷阱

1. **delete 操作**：创建空洞，降级为 HOLEY
2. **混合类型**：导致频繁的 Element Kind 转换
3. **稀疏数组**：V8 使用字典模式存储，性能大幅下降
4. **数组方法**：某些方法（如 map、filter）会创建新数组，注意内存分配

## 调试技巧

使用 V8 内置函数查看数组类型：
```javascript
const arr = [1, 2, 3];
console.log(%HasSmiElements(arr)); // true
console.log(%HasDoubleElements(arr)); // false
console.log(%HasObjectElements(arr)); // false
```

> 注意：需要启动 V8 时添加 `--allow-natives-syntax` 参数
